# سیاست بازبینی تغییرات (Review Policy)

> آخرین به‌روزرسانی: 2026-02-06

این سند حداقل قواعد بازبینی PR را برای Persian Tools تعریف می‌کند تا کیفیت، حریم خصوصی و پایداری مسیرهای اصلی حفظ شود.

## قواعد پایه

- هر PR باید یک Issue یا توضیح مساله مشخص داشته باشد.
- دامنه PR باید کوچک و متمرکز باشد؛ PRهای بسیار بزرگ باید شکسته شوند.
- نویسنده PR مسئول ارائه خروجی تست‌های واقعی است.

## معیار تایید PR

1. حداقل یک بازبین تایید کند.
2. gateهای کیفیت مرتبط پاس شوند.
3. چک‌لیست PR کامل شده باشد.
4. در صورت تغییر رفتار، مستندات مرتبط به‌روزرسانی شده باشد.

## سطح‌بندی ریسک و نیاز بازبین

- Low risk (docs/refactor محدود): یک بازبین.
- Medium risk (منطق ابزار/فرم‌ها): یک بازبین + تایید تست.
- High risk (PWA/Consent/Ads/Security): دو بازبین و الزام E2E مرتبط.

## موارد Blocker

- افزودن dependency خارجی runtime
- شکست تست‌های مرتبط بدون برنامه اصلاح
- رگرسیون واضح RTL/A11y در مسیرهای اصلی
- تغییرات حساس امنیتی بدون بررسی nonce/CSP/consent

## SLA پیشنهادی بازبینی

- پاسخ اولیه بازبین: حداکثر 1 روز کاری
- دور دوم بازبینی پس از اعمال اصلاحات: حداکثر 1 روز کاری

## چک‌لیست بازبین

- آیا تغییر با `docs/project-standards.md` سازگار است؟
- آیا Local-first و privacy-by-default حفظ شده است؟
- آیا تست‌های ادعاشده واقعاً اجرا و گزارش شده‌اند؟
- آیا مسیرهای RTL و a11y در دامنه تغییر بررسی شده‌اند؟
- آیا تغییر نیازمند به‌روزرسانی `docs/` بوده و انجام شده است؟
