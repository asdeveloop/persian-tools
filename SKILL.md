# مهارت مهندسی Persian Tools

این مهارت، دستورالعمل‌های عملیاتی و کیفیتی برای توسعه سریع اما امن و پایدار در Persian Tools است.

## اهداف کلیدی

- افزایش سرعت توسعه بدون افت کیفیت.
- یکپارچگی مستندات و کد.
- حفظ اصول local‑first، RTL و حریم خصوصی.

## اصول غیرقابل مذاکره

1. **Local‑first**: پردازش در مرورگر، بدون انتقال داده مگر با رضایت.
2. **بدون وابستگی خارجی در runtime**: عدم استفاده از CDN یا اسکریپت ثالث.
3. **RTL صحیح**: استفاده از کلاس‌های منطقی `start/end`.
4. **دسترس‌پذیری پایه (WCAG AA)** در مسیرهای اصلی.
5. **حریم خصوصی پیش‌فرض**: تحلیل‌گر/تبلیغات فقط با Opt‑in.

## مسیرهای اصلی پروژه

- `app/` روت‌ها و صفحات
- `components/` اجزای عمومی
- `features/` منطق ابزارها
- `shared/` هسته مشترک
- `public/` دارایی‌های self‑host

## فرآیند توسعه استاندارد

1. **مطالعه استانداردها**: `docs/project-standards.md` قبل از هر تغییر.
2. **یافتن محل تغییر**: جست‌وجو با `rg` و بررسی ساختار.
3. **پیاده‌سازی کوچک و قابل تست**.
4. **تست سریع**: `pnpm ci:quick`.
5. **به‌روزرسانی مستندات**: اگر رفتار تغییر کرد، `docs/` را هماهنگ کن.

## استاندارد کیفیت کد

- TypeScript `strict` فعال و بدون `any` غیرضروری.
- استفاده از توکن‌های طراحی در `shared/constants/tokens.ts`.
- رعایت `prefers-reduced-motion` در موشن‌ها.
- PWA: هر تغییر مهم، افزایش `CACHE_VERSION` در `public/sw.js`.

## تست و QA

- حداقل پوشش برای ابزارهای اصلی ≥ 85%.
- تغییرات PWA/Consent باید تست E2E داشته باشند.

## مستندات و هم‌راستایی

- فهرست کل مستندات: `docs/index.md`.
- نقشه راه: `docs/roadmap.md`.
- بورد مدیریتی: `docs/roadmap-board.html`.

## ضد الگوها (پرخطر)

- اضافه‌کردن اسکریپت خارجی در runtime.
- تغییر flow رضایت بدون به‌روزرسانی UI.
- استفاده از کلاس‌های LTR در صفحات RTL.

## خروجی مورد انتظار

- کد تمیز، ماژولار و قابل تست.
- مستندات به‌روز و هم‌خوان با رفتار واقعی.
- عملکرد پایدار و تجربه کاربری روان.
