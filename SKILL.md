# مهارت مدیریت پروژه (Refactored)

این مهارت برای خودکارسازی و مدیریت پروژه **Persian Tools** طراحی شده است. شما به عنوان مدیر پروژه، این مهارت را برای هدایت مدل Codex به منظور انجام مراحل مختلف توسعه استفاده خواهید کرد.

## مراحل پروژه

### 1. تولید کد

برای افزودن قابلیت جدید یا بهبود بخش‌های موجود در پروژه، کدهای لازم به طور خودکار تولید می‌شود. این کدها باید با استانداردهای پروژه تطابق داشته باشند.

**دستورالعمل‌ها**:

- کدها باید مطابق با استانداردهای `docs/project-standards.md` نوشته شوند.
- از معماری `docs/architecture.md` پیروی شود.
- هیچ وابستگی خارجی در زمان اجرا نباید وجود داشته باشد (CDN/فونت/اسکریپت خارجی ممنوع).
- UI فارسی و RTL با فونت IRANSansX.

### 2. مستندسازی

مستندات برای هر بخش از کد به طور خودکار تولید می‌شود.

**دستورالعمل‌ها**:

- مستندات باید شامل توضیحات دقیق برای توابع، ورودی‌ها، خروجی‌ها، و نحوه استفاده از آن‌ها باشد.

### 3. نوشتن و اجرای تست‌ها

تست‌های واحد و ادغام باید برای هر بخش از کد نوشته شده، به صورت خودکار اجرا شوند.

**دستورالعمل‌ها**:

- تست‌ها باید شامل منطق کسب‌وکار، سناریوهای خطا و دسترسی‌پذیری پایه باشند.
- ابزارهای تست: Vitest و Playwright در صورت نیاز.

### 4. رفع خطاها

در صورت شناسایی خطا، از ابزارهای خودکارسازی برای رفع این خطاها استفاده می‌شود.

**دستورالعمل‌ها**:

- از ابزارهای **Linting** و **Debugging** برای شناسایی و اصلاح مشکلات استفاده شود.

### 5. کامیت و ارسال کد

پس از تکمیل مراحل کدنویسی و تست، کد به صورت خودکار به مخزن گیت کامیت می‌شود.

**دستورالعمل‌ها**:

- پیغام‌های کامیت باید واضح و مطابق Conventional Commits باشند.

### 6. گزارش‌گیری و بررسی

گزارش دقیقی از وضعیت کد، تست‌ها و خطاهای شناسایی شده تولید می‌شود.

**دستورالعمل‌ها**:

- گزارش باید شامل اطلاعات مربوط به تست‌های موفق و ناموفق، تغییرات انجام شده و وضعیت مخزن باشد.

### 7. استفاده از API‌ها و پلتفرم‌ها

برای مدیریت اشتراک‌ها و تاریخچه، از API‌های مستند در `docs/api.md` استفاده می‌شود.

**دستورالعمل‌ها**:

- اطمینان حاصل شود که هیچ اطلاعات حساسی از کاربر به بیرون ارسال نمی‌شود مگر با رضایت صریح.
- برای مدیریت اشتراک‌ها و تاریخچه از `docs/api.md` تبعیت شود.

### 8. اسنپ‌شات و ادامه در چت جدید

قبل از رسیدن به حداکثر تعداد توکن‌ها، اسنپ‌شات از وضعیت کنونی گرفته شده و مراحل در چت جدید ادامه می‌یابد.

**دستورالعمل‌ها**:

- قبل از رسیدن به حداکثر توکن‌ها، اسنپ‌شات گرفته و در چت جدید مراحل ادامه می‌یابد.

## منابع

### ارجاعات

- **docs/project-standards.md**: مستندات استانداردهای پروژه.
- **docs/architecture.md**: مستندات معماری پروژه.
- **docs/api.md**: مرجع خلاصه API عمومی.
- **scripts/db/schema.sql**: اسکیمای دیتابیس اشتراک و تاریخچه.

### نکات اجرایی

- پوشه `docs` در `.gitignore` است مگر خلاف آن تنظیم شود.
- ابزارهای اجرای تست و lint از `package.json` فراخوانی شوند.
